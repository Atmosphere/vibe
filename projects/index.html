Getting involved

Getting involved is definitely the most dominant driving force of Vibe like any other open source project. Actually Vibe has started since someone who wrote Yet Another Atmosphere shared small experience as an effort to make Atmosphere the better project. There are lots of ways you can do.

Share experience

Do you have any idea or feel something goes wrong so want to start a war? Just tell us what you think via @vibe_project or Groups then we will respond. If you wrote some article about Vibe, that's perfect. Let us know your work and we will review if possible and link it. Also you can share knowledge by asking and answering questions on Groups and StackOverflow and can participate in conversations about issues on Groups and our GitHub repositories.

Report bugs

If you are sure it is a Vibe bug, browse relevant project's issue tracker and file an issue following the project's CONTRIBUTING.md if it has not reported. If you are not sure, post a question for that to Groups or StackOverflow instead to confirm it is indeed a bug in Vibe. If you have no time to dig it deeply, you don't need to show sincerity when posting a question. Though a lack of sincerity will result in low priority, it doesn't mean we will ignore it.

Commit your work

The fact is only a few people are willing to spend their precious time to do this kind of contribution for everyone. Hence we will never reject your effort for trivial reason like no test, code style, commit message and branch name as long as your work is reasonable. For those trivial things, we will ask you to fix them through review but if you are busy or it's annoying you, just tell us then we will fork, trim and push your work if you agree. Nevertheless, note that what you do defines you.

For a complicated and time consuming work like new feature, we need to discuss it on Groups first and then pull request of relevant GitHub repository enough for you to avoid wasting too much time and to lead Vibe to the desired way the community thinks.
